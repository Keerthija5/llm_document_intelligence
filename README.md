# LLM-Based Document Intelligence System

## 📌 Overview

This project focuses on building a **Document Intelligence System using Large Language Models (LLMs)** to process unstructured business text and convert it into structured insights.

The system is designed to handle real-world document types such as:

* support tickets
* maintenance reports
* meeting notes

The main goal is to reduce manual effort in reading raw text and transform it into useful outputs such as summaries, action items, document labels, and model evaluation results.

---

## 🎯 Why This Project Matters

In many real-world workflows, teams spend time manually reading documents, identifying important information, and converting that information into actions. This project shows how LLMs can help automate that process by turning unstructured text into structured and usable insights.

This makes the project relevant for document-heavy workflows in areas such as:

* operations
* maintenance
* support
* internal reporting

---

## 🚀 What I Implemented

* Designed an end-to-end document processing pipeline
* Implemented LLM-based summarization using Hugging Face models
* Compared two transformer models (**BART** and **T5**) for summary generation
* Built a rule-based evaluation system to compare output quality
* Developed NLP-based extraction for action items and keywords
* Added document-level classification for category, priority, and urgency
* Generated structured outputs in **JSON** and **CSV** formats

---

## ⚙️ Workflow

```text
Input Text → Summarization (BART & T5) → NLP Processing → Evaluation → Output (JSON + CSV)
```

---

## 🤖 Model Comparison

| Model    | Observation                                         |
| -------- | --------------------------------------------------- |
| **BART** | More fluent and structured summaries                |
| **T5**   | Sometimes longer, but less consistent in formatting |

The evaluation system compares both models based on:

* sentence completeness
* formatting quality
* readability
* structure consistency

---

## 📈 Results & Observations

* BART generally produced more fluent and cleaner summaries
* T5 sometimes generated longer outputs, but formatting was less consistent
* The evaluation step helped compare outputs systematically instead of choosing a model manually
* The project showed that even when both models perform reasonably well, structured evaluation is important for selecting the better output
* The pipeline successfully converted raw business-style text into structured insights
